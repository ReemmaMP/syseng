Why did we choose Xamarin?

Since the requirement was to create a cross-platform app, we had the choice of using either Xamarin or an HTML-wrapping framework such as Apache Cordova or PhoneGap. There were a few factors that pushed us toward Xamarin:

  • As this is a Microsoft-sponsored project, use of the Microsoft stack is encouraged (and preferred) by our clients.
  • Last year's work on this project was done in C# and Xamarin so it made sense to use this technology again in order to be able to re-use any code where possible
  • Writing the app in C# allows us to build a rich, native application rather than one that relies on an embedded web browser. This yields a UI that feels more native to the platform, complete with backend code re-use, as opposed to a Cordova app that looks the same on all platforms even if that breaks each given platform's design guidelines.
  • The application uses lots of complex APIs like location tracking which can be more challenging and less reliable in pure JavaScript. Xamarin promises 100% API access from within C#, so our code would even be able to access, for example, Bluetooth Low Energy in the future if it were to be combined with the work of another Spot Markets group.

How to build, setup and configure the platform?

  • All code we write will be in Microsoft Visual Studio, so this will need to be installed.
  • Code is hosted on Visual Studio Team Services, so obtaining the code, should access be granted, can be done using either Git or the Team Services pane in Visual Studio itself.
  • Once the code is cloned, there will be a single solution (.sln) file which contains all the code for our main application. There may also be other code in the repository if we carry out any further experiments and API tests, but the main repo will be clearly labelled.
  • When opening the project for the first time, all the required NuGet packages will be restored to the project.
  • Choose the platform you want to build for from the dropdown list in Visual Studio. If you are building for iOS you should have a Mac available with Xcode and the Xamarin agent installed to handle builds.

The following technologies are in use:

  • Recommendation Engine (not being developed by us, but underpins the machine learning aspects of the project): Azure and AzureML.
  • Database: Azure DocumentDB accessed via the MongoDB API.
  • API: JSON is used to communicate between the client and the server.
  • Flask: The backend server is written in Flask / Python. This is being deployed to a Linux virtual machine hosted on Azure Compute.
  • Bing Maps API: We plan to use Bing Maps to show the user where they are and to overlay the Points of Interest onto a map.
  • OpenWeatherMap API: We are using OpenWeatherMap (as it is free up to one call per second) to source local weather data into the app.
