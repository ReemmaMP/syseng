Testing strategies

Along with researching about how to build the app, we looked into Unit testing as well as this is essential to ensuring that our code works in every possible scenario that can be generated. According to this article*, our unit test should be equipped to do the following:

  • - Set the conditions (pre-conditions)
  • - Cause the trigger
  • - Check for the expected results

Due to the time limit on this project, we may not be able to explore all the different options for testing. However, a few may be more relevant and good to have implemented. We will be focusing on testing for most commonly occurring issues. For example, we may find that boundary conditions* are a prime source of the bugs found in our application code. Similar to the method shown, we could implement a unit test to catch a StackOverflowException, and if it’s not caught, then the unit test would fail.

In addition to this, we looked at Visual Studio specific guides on how to set up Unit Tests. As we were still at research stage, we only practised unit testing on smaller, dummy apps we made. To make it work on our Xamarin application, we would need to do the following**:

  • - Create a test project
  • - Create a test methods within the projects
  • - Write the testsSet timeout on each method using a unit test framework. We found out that Visual Studio IntelliSense will be a useful guide in doing this
  • - Set timeout on each method
  • - Practise running the tests in a test explorer and check for passes and fails and outstanding tests
