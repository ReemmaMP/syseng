Design Patterns


  • Model View Controller- Our Django webapp for Kent County Council is built following the Model View Controller architectural pattern. The Model part corresponds to our permanent store for all the data we collect through our webapp, which is the access to our Postgresql database. The View is in charge of the User Interface logic of the application as it generates the HTML code and produces its output which is what the user is able to see. The Controller is the interface between the Model and the View and in our webapp, it is responsible for updating the Model based on user input in the View. [1] It is hosted in an Azure instance.

  • Singleton Pattern- Singleton Design Pattern is one of the Gang of Four design patterns we have used to only a single instance of the singleton class is created. One of the ways in which we were able to implement it in C# is by creating a static variable which then holds a reference single instance that has been created [2].
